# tLeap Guide: Building AMBER Topology Files

## What is tLeap?

**tLeap** (Toolkit LEap) is an AMBER tool that:
- Reads molecular structures (PDB, MOL2 formats)
- Applies force field parameters (bonds, angles, dihedrals, charges)
- Generates AMBER topology files (.top) and coordinate files (.crd)
- Performs sanity checks on the structure

**Purpose:** Convert parametrized molecular structures into AMBER-compatible format, which can then be converted to LAMMPS or GROMACS formats.

## Step-by-Step: The tLeap Script

### Complete tLeap Input File (tleap.in)

```bash
source leaprc.gaff2
SUS = loadmol2 PVA7_trim.mol2
check SUS
loadamberparams PVA7_trim.frcmod
saveamberparm SUS PVA7_trim.top PVA7_trim.crd
quit
```

### Line-by-Line Explanation

#### Line 1: Load Force Field
```bash
source leaprc.gaff2
```

**What it does:**
- Loads the GAFF2 force field database
- Makes available all GAFF2 atom types, bond parameters, angle parameters, dihedral parameters, and Lennard-Jones parameters
- Initializes tLeap with the GAFF2 environment

**What it contains:**
- Atom types: c3, oh, hc, etc.
- Van der Waals parameters (Lennard-Jones)
- Default bond/angle parameters
- Default charges (before Antechamber corrections)

**Why GAFF2?**
- General AMBER Force Field v2
- Broad applicability to organic molecules
- Automated atom typing and parameter assignment

---

#### Line 2: Load Molecular Structure
```bash
SUS = loadmol2 PVA7_trim.mol2
```

**What it does:**
- Reads the MOL2 file (MOL2 = SYBYL MOL2 format)
- Parses all atoms, bonds, charges, and atom types
- Assigns this structure to a variable named `SUS` (arbitrary name)

**What `SUS` is:**
- A tLeap unit (molecular structure object)
- Contains: all atoms, bonds, atom types, partial charges
- Can be referred to by name in subsequent commands

**Input file:** `PVA7_trim.mol2`
- Generated by Antechamber in the previous step
- Contains: PVA structure with GAFF2 atom types and Antechamber-assigned charges
- Format: MOL2 (human-readable, column-based)

**Example MOL2 content:**
```
@<TRIPOS>MOLECULE
PVA7_trim
 70 69 0 0 0
TEMP_NAME
NO_CHARGES

@<TRIPOS>ATOM
      1 C1         0.0000    0.0000    0.0000 c3     1  PVA7   0.0980
      2 H1         0.8900    0.0000    0.0000 hc     1  PVA7  -0.0230
      3 H2        -0.4450    0.8160    0.0000 hc     1  PVA7  -0.0230
      4 C2         1.5400    0.0000    0.0000 c3     1  PVA7  -0.0520
      ...
```

#### Line 3: Validation Check
```bash
check SUS
```

**What it does:**
- Performs sanity checks on the structure
- Checks for:
  - Missing atoms or bonds
  - Unusual connectivity
  - Missing parameters for specific atom types
  - Charge neutrality issues

**Output:** Prints to leap.log
- If successful: "No Problems found."
- If issues: Detailed warnings about missing parameters

**Example output:**
```

	-- There are 0 impropers

Checking for bond parameters.
Checking for angle parameters.
Checking for dihedral parameters.
Done checking SUS (SUS).
```

#### Line 4: Load Custom Parameters
```bash
loadamberparams PVA7_trim.frcmod
```

**What it does:**
- Reads the force field modification file (.frcmod)
- Adds/overrides parameters for specific atom type combinations
- Fills in gaps left by the default GAFF2 database

**What .frcmod contains:**
- Bond parameters (force constant, equilibrium distance)
- Angle parameters (force constant, equilibrium angle)
- Dihedral parameters (dihedral barrier heights)
- Improper parameters (for out-of-plane motion)

**Example .frcmod content:**
```
bond parameters
c3 c3   222.0   1.5380
c3 oh   320.0   1.4100
oh ho   553.0   0.9600

angle parameters
c3 c3 c3    63.0   109.47
c3 c3 oh    67.0   109.47
c3 oh ho    55.0   108.50

dihedral parameters
X  c3 c3  X      1    0.1556 180.0 -2.0
X  c3 oh  X      1    0.2000 180.0 -2.0
```

**Why needed?**
- Antechamber assigns charges
- Parmchk2 fills in missing parameters
- .frcmod contains the filled-in parameters for this specific molecule
- Without it, tLeap would use generic GAFF2 defaults (may be less accurate)

---

#### Line 5: Save Topology and Coordinates
```bash
saveamberparm SUS PVA7_trim.top PVA7_trim.crd
```

**What it does:**
- Writes AMBER topology file (.top)
- Writes AMBER coordinate file (.crd)
- Both contain fully parametrized system

**Output files:**

**PVA7_trim.top (Topology File):**
- Contains: Force field parameters for all atoms
- Lists: All bonds, angles, dihedrals, impropers
- Includes: Atom names, types, masses, charges
- Used by: AMBER, or can be converted to LAMMPS/GROMACS

**Example .top content:**
```
%FORMAT(20a4)
PVA7 created by LEaP

%FLAG TITLE
%COMMENT
 PVA7 created by LEaP
%FORMAT(20a4)

%FLAG POINTERS
%FORMAT(10I8)
     70     69      0      0      0      0      0      0      0      0
...
```

**PVA7_trim.crd (Coordinate File):**
- Contains: Atomic coordinates from MOL2
- Simple format: one line per atom with x, y, z
- Can be viewed with text editor

**Example .crd content:**
```
PVA7 created by LEaP
    70
   1  0.0000  0.0000  0.0000
   2  0.8900  0.0000  0.0000
   3 -0.4450  0.8160  0.0000
...
```

#### Line 6: Exit tLeap
```bash
quit
```

**What it does:**
- Exits tLeap gracefully
- Saves all work to output files
- Writes summary to leap.log

## Troubleshooting tLeap

### Problem: "Could not open file leaprc.gaff2"

**Cause:** GAFF2 force field database not found

**Solution:**
```bash
# Verify AmberTools is installed
which tleap

# Check AMBERHOME is set
echo $AMBERHOME

# If not set, set it:
export AMBERHOME=/path/to/amber
export PATH=$AMBERHOME/bin:$PATH
```

### Problem: "Unsupported atom type"

**Cause:** MOL2 file has atom type not in GAFF2 database

**Solution:**
1. Check Antechamber output for warnings
2. Verify MOL2 file has correct atom types:
   ```bash
   grep "c3\|oh\|hc" PVA7_trim.mol2 | head -5
   ```
3. Re-run Antechamber with correct parameters

### Problem: "Could not open file PVA7_trim.frcmod"

**Cause:** File doesn't exist or wrong filename in tleap.in

**Solution:**
```bash
# Verify file exists
ls -la PVA7_trim.frcmod

# Check tleap.in has correct filename
grep frcmod tleap.in
```

### Problem: "Charge mismatch"

**Cause:** Total system charge is not an integer (should be close to 0)

**Cause:** Happens if:
- Antechamber miscalculated charges
- Partial charges are missing

**Solution:**
```bash
# Check charges in MOL2 file
grep -A 1000 "@<TRIPOS>ATOM" PVA7_trim.mol2 | grep -v "^$" | tail -5

# Calculate total charge:
python3 -c "
import numpy as np
charges = [...]  # list of charges from MOL2
print(f'Total charge: {sum(charges):.5f}')
"
```

## How tLeap Creates Parameters

### Bond Parameters Example

**From leaprc.gaff2 (default):**
```
c3-c3 bond:   222.0 kcal/(mol·Å²)   r₀ = 1.5380 Å
```

**From PVA7_trim.frcmod (custom):**
```
c3 c3   222.0   1.5380
```

**tLeap uses:** The custom value (from .frcmod) because it's more specific

### Dihedral Parameters Example

**From leaprc.gaff2:**
```
X-c3-c3-X dihedral: V/2 = 0.1556 kcal/mol
```

**Applied to:** Any c3-c3 bond in any context

**Custom dihedrals in .frcmod:** Override for specific atom environments

## Output File Formats

### AMBER Topology File (.top)

**Contains:**
- Atom information (names, types, masses, charges)
- Bond definitions (which atoms are bonded)
- Angle definitions (which atoms form angles)
- Dihedral definitions
- Non-bonded parameters
- Force field constants

**Can be used with:**
- AMBER (directly)
- LAMMPS (after conversion with amber_to_lammps.py)
- GROMACS (after conversion)

### AMBER Coordinate File (.crd)

**Contains:**
- Title line
- Number of atoms
- Atomic coordinates (x, y, z per atom)
- Box dimensions (if periodic)

**Format:**
```
PVA7 created by LEaP
    70
   1  0.000000  0.000000  0.000000
   2  0.890000  0.000000  0.000000
   ...
```

## Key Takeaways

1. **tLeap is the AMBER topology builder**
   - Converts parametrized MOL2 files to AMBER topology format

2. **The script has 5 key steps**
   - Load force field (leaprc.gaff2)
   - Load structure (loadmol2)
   - Validate (check)
   - Load custom parameters (.frcmod)
   - Save output (saveamberparm)

3. **Usage:** `tleap -f tleap.in`
   - Reads commands from input file
   - Creates .top and .crd output files
   - Generates leap.log for debugging

4. **Output is AMBER format**
   - Can be used directly with AMBER
   - Can be converted to LAMMPS or GROMACS
   - Next step in full workflow

5. **Always check leap.log**
   - Look for warnings or errors
   - Verify charge neutrality
   - Confirm all parameters were loaded
