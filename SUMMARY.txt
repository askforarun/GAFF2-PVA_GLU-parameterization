================================================================================
THE THREE PARAMETRIZATION STEPS
================================================================================

STEP 1: Build PVA with Hard-Coded Geometry
──────────────────────────────────────────
Module: pva_builder.py
Function: build_pva(n, output_file, cap=False)

Input:  Chain length n (e.g., 7 monomers)
Output: PVA{n}_trim.pdb (uncapped PVA structure)

Description:
  - Builds PVA polymer: CH3-(CH2-CHOH-CH2)n-CH3
  - Uses hard-coded bond lengths (C-C ~1.54Å, C-O ~1.42Å, C-H ~1.09Å)
  - Uses tetrahedral angles (109.47°)
  - NOT minimized (scaffold for parametrization)
  - Ready for Antechamber parametrization


STEP 2: Parametrize with GAFF2 (Antechamber + Parmchk2)
───────────────────────────────────────────────────────
Module: molecular_utils.py
Function: getfiles(pdb_file)

Input:  PVA{n}_trim.pdb
Outputs: 
  - PVA{n}_trim.mol2     (MOL2 format with GAFF2 atom types)
  - PVA{n}_trim.frcmod   (Force field corrections)
  - PVA{n}_trim.top      (AMBER topology)
  - PVA{n}_trim.crd      (AMBER coordinates)

Description:
  - Runs Antechamber: assigns GAFF2 atom types, initial charges
  - Runs Parmchk2: fills in missing FF parameters (bonds, angles, dihedrals)
  - Runs tLeap: generates AMBER topology file
  - Includes hand-corrections: ha→hc, c2→c3 (proper aliphatic types)
  - Requires: AmberTools (conda install -c conda-forge ambertools)


STEP 3: Assign Pre-Extracted Partial Charges
──────────────────────────────────────────────
Module: molecular_utils.py
Function: load_system_charges(chain_length, n_pva, n_glu)

Inputs: Chain length, number of PVA chains, number of GLU molecules
Output: numpy array of partial charges (1-D)

Reference Files Used:
  - charge_data/PVA_monomercharges.txt      (10 atoms per monomer)
  - charge_data/glutaraldehyde_charges.txt  (18 atoms per molecule)

Description:
  - Loads pre-extracted charges from minimized reference structures
  - Tiled across all chains/molecules in the system
  - Ensures consistency across all jobs in workflow
  - Charges extracted from minimized structures (PVA7_min.pdb)
  - Avoids re-running Antechamber per job

Why Pre-Extracted Charges?
  ✓ Consistent across all jobs
  ✓ From minimized reference (chemically sound)
  ✓ Avoids computational redundancy
  ✓ Improves reproducibility

================================================================================
FORCE FIELD INFORMATION
================================================================================

Force Field: GAFF2 (General AMBER Force Field v2)

Characteristics:
  - Broad applicability to organic molecules
  - Automated parameter assignment via Antechamber
  - Includes bonds, angles, dihedrals, impropers
  - Lennard-Jones van der Waals parameters
  - Electrostatic terms (partial charges)

Reference:
  Wang et al., J. Comp. Chem. 2004
  http://ambermd.org/

For PVA specifically:
  - Atom types: c3 (sp3 carbon), oh (hydroxyl oxygen), hc (alkane hydrogen)
  - Hand-corrected from Antechamber defaults (ha→hc, c2→c3)
  - Parametrization includes standard C-C, C-O, C-H bonds
  - Dihedral corrections ensure proper rotamer barriers
